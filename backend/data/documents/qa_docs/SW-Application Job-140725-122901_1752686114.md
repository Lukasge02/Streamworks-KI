# SW-Application Job-140725-122901\n\nApplication Job Application Job (AppJob) Application Jobs (kurz AppJob) wurden im Streamworks Release R9.3 eingeführt. Im Release R9.3 wurde die grundlegende Funktionalität der AppJobs integriert. Der weitere Ausbau der Funktionalität erfolgt in den folgenden Releasen. AppJobs erleichtern Streamworks Anwendern das Anlegen von komplizierten Jobs oder von größeren Jobnetzen (in R9.3 nur sequentielle Jobnetze), die eine komplizierte Verarbeitung abbilden. Wichtige Links Ablageort produktiv eingesetzter AppJobs => Produktive AppJobs Technische Entwicklerdokumentation => Entwickler Dokumentation (OneNote) Master Data AppJobs werden im Streamworks System pro Mandant unter Mandant\Configurations\Application Jobs hinterlegt. In der Funktion Application Jobs werden alle AppJobs des Mandanten mit Name, Autor, Version und Status angezeigt Neue AppJobs werden mit der Kontext Funktion "Add Application Job" eingefügt\n\nIn der Funktion Application Jobs werden alle AppJobs des Mandanten mit Name, Autor, Version und Status angezeigt Neue AppJobs werden mit der Kontext Funktion "Add Application Job" eingefügt AppJobs können aktuell per GUI nur vor dem ersten Speichern wieder aus der Liste gelöscht werden. Danach müssen sie manuell aus der DB entfernt werden. AppJobs mit gleichem Namen und gleicher Version können nicht mehrfach eingefügt werden. AppJobs können aktiv oder inaktiv sein. Inaktive AppJobs werden bei der Selektion im Job nicht angezeigt. Stream Design Ein Job im Stream Design wird zu einem AppJob wenn man als Job Template Type den Wert Application selektiert. Nun wird in den Job Eigenschaften (Job Properties) der Bereich Application Job mit den Eigenschaften Type und Settings angezeigt. Unter Type kann jetzt ein beliebiger Application Job aus einer Liste ausgewählt werden, welche anhand der Einträge in der Master Data Funktion Application Jobs erstellt wird.\n\nAnschließend kann durch den Button Ok die Auswahl bestätigt werden oder durch den Button Next direkt mit der Eingabe der Settings begonnen werden. Beispiel für einen Settings Dialog. Nachdem die Type und Settings definiert worden sind ist der Application Job fertiggestellt. Aufbereitung und Runtime Im Zuge der Aufbereitung wird das beim AppJob hinterlegte Jobnetz (1-n Jobs) in den Stream an der Stelle des AppJobs eingefügt. In der Runtime existiert somit nicht mehr der AppJob sondern nur neu integrierten Jobs. Die Jobnamen der neu integrierten Jobs werden gebildet aus: Beispiel: Job Name Stream Design: "0010_job"; Job Kürzel aus der AppJob Definition "_AJ01" => Job Name in der Runtime: "0010_job_AJ01" In folgendem Beispiel sieht man, dass aus einem AppJob heraus 8 Jobs generiert wurden.
